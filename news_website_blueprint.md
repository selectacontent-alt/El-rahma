# مخطط موقع أخبار للشركة — Light Mode

**الهدف:** بناء موقع أخبار عربي خاص بالشركة، مربوط بالدومين الأساسي، بتصميم جذاب مستوحى من النماذج المرسلة، ويخدم تحسين الظهور في محركات البحث SEO من خلال محتوى متجدد، بنية أقسام واضحة، وروابط داخلية قوية.

**الألوان الأساسية:**

- اللون الرئيسي: `#9d027c`
- اللون المساعد / الأكشن: `#ffbc01`
- الخلفية العامة: `#f7f7f9`
- خلفية البطاقات: `#ffffff`
- النص الأساسي: `#1f1f1f`
- النص الثانوي: `#666666`
- حدود خفيفة: `#e8e8ee`

---

## 1. الفكرة العامة للموقع

الموقع يكون **بوابة أخبار ومقالات للشركة**، الهدف منها نشر أخبار المجال، تحديثات الشركة، مقالات تعليمية، أدلة، ترندات، فيديوهات، بودكاست، ودراسات حالة. التصميم يكون عربي بالكامل RTL، لايت مود، سريع، منظم، وسهل القراءة.

الشكل العام مستوحى من الصور المرسلة:

- هيدر واضح فيه شعار الشركة، قائمة أقسام، بحث، وروابط سوشيال.
- بانر إعلاني أو تعريفي أعلى الصفحة.
- شريط أخبار عاجلة أو أحدث الأخبار.
- قسم رئيسي Hero فيه خبر كبير بارز مع أخبار صغيرة بجانبه.
- تقسيم الصفحة إلى أقسام كثيرة لكن منظمة.
- بطاقات أخبار بصورة وعنوان ومعلومات مختصرة.
- عمود جانبي للترندات، آخر الأخبار، الإعلانات، والنشرة البريدية.
- فوتر كبير فيه روابط مهمة وسوشيال وخريطة الموقع.

## 2. هوية التصميم UI Direction

### الطابع البصري

- تصميم حديث ومرتب، ليس مزدحمًا مثل المواقع القديمة.
- مساحات بيضاء كثيرة لراحة العين.
- استخدام اللون البنفسجي `#9d027c` للعناوين، الخطوط الفاصلة، الأزرار الرئيسية، والأيقونات المهمة.
- استخدام الأصفر `#ffbc01` للتنبيهات، الأخبار العاجلة، الوسوم، أزرار الـ CTA، وأيقونات التمييز.
- الكروت تكون بخلفية بيضاء وظل خفيف جدًا.
- الصور بزوايا دائرية بسيطة `8px - 14px`.
- العناوين Bold وواضحة.
- كل قسم له عنوان واضح مع خط صغير بالألوان الأساسية.

### الخطوط المقترحة

- العناوين: **Cairo Bold** أو **Tajawal Bold**
- النصوص: **Tajawal Regular** أو **Cairo Regular**
- الأرقام والتواريخ: **IBM Plex Sans Arabic** أو **Cairo**

## 4.5 Hero Section

**هذا أهم جزء في الصفحة.**

- يمين أو منتصف: خبر رئيسي كبير بصورة كبيرة.
- بجانبه: 2 أو 4 أخبار صغيرة.
- عمود جانبي: الأكثر قراءة / ترند اليوم.

```text

│       Big Image + Title     │ Small Article    │ 1. خبر ترند      │
```

- صورة كبيرة بنسبة 16:9.
- Overlay خفيف Gradient من الأسفل.
- Badge للقسم مثل “أخبار الشركة”.
- عنوان كبير واضح.
- تاريخ + اسم الكاتب.

### ألوان Hero

- الـ Badge باللون `#ffbc01`.
- العنوان أبيض فوق الصورة.
- إطار أو خط علوي صغير باللون `#9d027c`.

# 6. خطة SEO كاملة للموقع

## 6.1 بنية الروابط URL Structure

استخدم روابط قصيرة وواضحة:

```text
/company-news/new-service-launch/
/articles/how-to-improve-seo/
/category/digital-marketing/
/case-studies/client-growth-case-study/
/videos/social-media-tips/
```

## 6.2 Meta Tags لكل صفحة

كل صفحة يجب أن تحتوي على:

- Meta Title لا يزيد عن 60 حرفًا.
- Meta Description بين 140 و 160 حرفًا.
- Canonical URL.
- Open Graph للسوشيال.
- Twitter Card.

## 6.3 Schema Markup

يتم إضافة Schema حسب نوع الصفحة:

- `Organization`
- `WebSite`
- `NewsArticle`
- `Article`
- `BreadcrumbList`
- `VideoObject`
- `FAQPage`
- `LocalBusiness` لو الشركة لها مقر محلي.

## 6.4 الربط الداخلي Internal Linking

كل مقال يجب أن يربط إلى:

- مقالين مرتبطين.
- صفحة خدمة واحدة على الأقل.
- صفحة تواصل معنا عند وجود CTA.
- قسم مرتبط.

مثال:

مقال عن SEO يربط إلى:

- خدمة تحسين محركات البحث.
- مقال عن الكلمات المفتاحية.
- دراسة حالة عن رفع الزيارات.

## 6.5 محتوى مستمر

خطة نشر مقترحة:

- 3 أخبار قصيرة أسبوعيًا.
- 2 مقالات تعليمية أسبوعيًا.
- 1 دراسة حالة شهريًا.
- 2 فيديو قصير شهريًا.
- تحديث المقالات القديمة كل 3 شهور.

## 6.6 عناصر تزيد ثقة جوجل E-E-A-T

- صفحة كاتب لكل محرر.
- نبذة عن الكاتب أسفل المقال.
- تاريخ النشر وآخر تحديث.
- مصادر وروابط موثوقة عند الحاجة.
- صفحة من نحن قوية.
- بيانات تواصل حقيقية.
- سياسة خصوصية وشروط استخدام.

## 6.7 سرعة الموقع Core Web Vitals

- الصور بصيغة WebP.
- Lazy Loading للصور.
- Minify CSS و JS.
- استخدام CDN.
- كاش قوي.
- عدم تحميل إعلانات ثقيلة.
- تجنب Sliders كثيرة وثقيلة.

# 7. تجربة الموبايل Mobile Layout

على الموبايل:

- الهيدر يتحول إلى Logo + Menu Icon + Search.
- الـ Breaking News يظهر كسطر واحد قابل للتمرير.
- Hero يصبح خبر كبير ثم الأخبار الصغيرة تحته.
- Grid الأخبار يصبح عمود واحد.
- Sidebar ينتقل أسفل المحتوى.

# 13. ملاحظات مهمة قبل التنفيذ

1. الموقع لا يكون نسخة من المواقع المرسلة، لكنه يستلهم منها فكرة البوابة الإخبارية المقسمة لأقسام.
2. التصميم يجب أن يكون أخف وأنظف من النماذج حتى يعطي إحساس شركة محترفة.
3. لا يتم الإكثار من الإعلانات داخل الصفحة حتى لا يضر تجربة المستخدم وSEO.
4. كل صورة يجب أن يكون لها Alt Text وصيغة WebP.
5. لازم يكون فيه CMS لإضافة الأخبار بسهولة.
6. الأفضل استخدام Next.js أو WordPress حسب فريق التطوير.
7. الموقع يفضل يكون داخل دومين الشركة مثل:

```text
https://domain.com/news/
```

أو Subdomain مثل:

```text
https://news.domain.com/
```

لكن الأفضل للـ SEO غالبًا أن يكون داخل نفس الدومين في مجلد `/news/` حتى يستفيد الدومين الأساسي من المحتوى والروابط الداخلية.
